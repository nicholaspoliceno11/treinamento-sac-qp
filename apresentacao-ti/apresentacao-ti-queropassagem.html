Documento preparado para apresentação ao gerente de TI, com visão de arquitetura, custos, segurança, riscos e melhorias do portal de treinamento do SAC.

1. O que é o projeto

Portal de treinamento do SAC com conteúdo em Markdown, login por perfil (Atendente, Backoffice, Administrador), acompanhamento de progresso, informativos internos, quiz (Desafio do Dia), chatbot Mila e gestão de usuários pelo admin.

Tela de login com a agente virtual Mila

Página inicial — hero, progresso e informativos
Menu lateral colapsável por tópicos
Página de treinamento (ex.: Atendimento)
Central de Dúvidas — fórum interno

4. Pontos positivos

  • Custo muito baixo para o volume atual (Firebase free tier + Google Workspace)
  • Feito sob medida para o fluxo do SAC da Quero Passagem
  • Segurança implementada: hash de senha, token de sessão (12h), anti brute-force, troca obrigatória no 1º acesso
  • Gestão de usuários pelo próprio portal (admin não edita planilha manualmente)
  • Conteúdo ágil: treinamentos em Markdown; informativos e FAQ da Mila pelo admin
  • Deploy automatizado via GitHub → Firebase
  • Sem servidor próprio para manter
  • Já em produção e validado pela operação

6. Pontos de atenção / riscos

  • Hoje roda em conta Google pessoal — risco se a pessoa sair
  • Google Sheets como banco — limitado para escala muito alta e auditoria pesada
  • Apps Script tem quotas (requisições e tempo de execução)
  • Sem SSO / login Google corporativo ainda
  • Deploy do backend exige publicar nova versão no Apps Script manualmente
  • Dependência do ecossistema Google
  • Sem ambiente de homologação formal

7. Perguntas frequentes do TI

Por que não usar um LMS pronto (Moodle, etc.)?

Porque o portal é leve, customizado para o SAC, com custo quase zero e atualização rápida. É biblioteca operacional, não LMS genérico.

O que precisa ser liberado no Google corporativo?

Conta de serviço Workspace, Apps Script (Web App), Google Sheets, Google Drive, Firebase Hosting e opcionalmente domínio customizado (ex.: treinamento.queropassagem.com.br).

As senhas ficam seguras?

Sim: hash SHA-256 com sal, token de sessão, bloqueio após 5 tentativas, troca obrigatória no primeiro acesso e regras de senha forte.

Apps Script como "Qualquer pessoa" é seguro?

Sim — é configuração obrigatória de Web Apps públicas. A segurança vem do login + token + perfil validados no servidor.

E LGPD?

Há dados pessoais (nome, e-mail, progresso). Migrar para Workspace corporativo facilita contrato/DPA, backups e controle de acesso.

Aguenta quantos usuários?

Dezenas a algumas centenas com uso típico de treinamento. Para escala muito maior, evoluir backend para Cloud Run + banco relacional.

Dá para SSO com Google da empresa?

Ainda não, mas é melhoria viável com Google OAuth / Workspace.

8. Melhorias recomendadas

Curto prazo (produção corporativa)

  • Migrar planilha, Apps Script, Firebase e Drive para conta Google Workspace
  • Conta de serviço (não pessoal) como dona dos recursos
  • Domínio corporativo + política de backup da planilha

Médio prazo

  • SSO com Google Workspace
  • Ambiente de homologação
  • Relatórios de progresso por equipe
  • Log de auditoria administrativa

9. O que solicitar ao TI

  1. Conta Google Workspace para o projeto (ex.: treinamento@empresa)
  2. Transferência da planilha e Apps Script para essa conta
  3. Projeto Firebase corporativo ou transferência de ownership
  4. DNS para domínio do treinamento
  5. Política de acesso mínimo à planilha (só admins editam)
  6. Confirmação de Apps Script e Drive nas políticas do Workspace
Mensagem de fechamento: O portal já funciona em produção com custo quase zero. O pedido ao TI é formalizar a infraestrutura no domínio Google da empresa — conta corporativa, backups, domínio e políticas — para sair da conta pessoal com segurança.
